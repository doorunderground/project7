# RC카 자율주행 프로젝트

## 프로젝트 개요

라즈베리파이 기반 RC카가 트랙 위의 차선을 인식하고 **카메라 한 개만으로** 자율주행하는 시스템입니다.

| 항목 | 내용 |
|------|------|
| 플랫폼 | 라즈베리파이 + YAHBOOM RC카 |
| 카메라 | Picamera2 (640×480) |
| 학습용 PC | Windows 11, GPU 탑재 |
| 통신 | PC ↔ 라즈베리파이 UDP |

### 트랙 특징
- 검정 아스팔트 위에 **노란 실선(외곽)** + **흰색 점선(중앙)** 으로 구성
- S자 커브·직선 구간 혼합, RC카가 겨우 지나갈 정도로 좁은 폭
- YAHBOOM 브랜드 트랙 매트

---

## 왜 모방학습을 선택했는가?

처음에는 차선 세그멘테이션 결과만으로 핸들 제어값을 계산하려 했으나 한계가 있었습니다.

| 방식 | 문제점 |
|------|--------|
| 차선 세그멘테이션만 사용 | 마스크 위치 → 조향각 변환이 불안정, S자 커브 대응 어려움 |
| 강화학습 | 실제 RC카 환경에서 시뮬레이터 없이 탐색 비용이 너무 큼 |
| **모방학습 (채택)** | 사람이 직접 운전한 데이터를 그대로 학습 → 안정적인 기본 주행 확보 |

> **모방학습이란?** 강화학습처럼 보상을 설계하고 스스로 탐색하는 것이 아니라,
> 사람의 시연 데이터(정답 행동)를 직접 모사하는 방식입니다.

**최종 채택 전략: 모방학습 + 차선 안전장치 조합**

```
모방학습 → 기본 주행 (코너·직선 모두 학습된 대로 판단)
차선 안전장치 → 모방학습이 틀렸을 때 강제 보정 (안전망)
```

```
  모터 제어 ◄────UDP 5001──────────      │                              │

```

## 구현 1 : 기본 자율주행 (모방학습 + 차선 안전장치)

### 1단계 - 모방학습 데이터 수집

| 항목 | 내용 |
|------|------|
| PC 코드 | `Pc_server(wasd).py` |
| 라즈베리파이 코드 | `Pi_drive(wasd).py` |
| 저장 경로 | `C:/PROJECT7/drive_dataset/` |
| 테스트 | `test_auto_drive(wasd).py` |

**동작 흐름**
1. 라즈베리파이가 카메라 영상을 JPEG로 압축 → UDP로 PC에 전송
2. PC에서 영상을 화면에 표시, 사람이 WASD로 RC카 원격 조종
3. R키를 누르면 녹화 시작 → `프레임.jpg + 프레임.txt(모터값, 키조합)` 저장
4. stop(아무 키도 안 누름) 구간은 저장 제외 → 클래스 불균형 방지

```
저장 파일 예시:
  frame_000123.jpg   ← 카메라 영상
  frame_000123.txt   ← "-80,-20\nwd"  (모터값 / 키조합)
```

**WASD 키 → 모터 매핑**
```
W      → 전진        W+A → 전진+좌
S      → 후진        W+D → 전진+우
A      → 좌회전      S+A → 후진+좌
D      → 우회전      S+D → 후진+우
```

> **동시 입력 처리 방법**: `cv2.waitKey()`는 단일 키만 감지하므로,
> 각 키에 만료 시각을 부여해 0.15초 내 눌린 키를 모두 활성으로 취급합니다.

---

### 2단계 - 모방학습 모델 학습

| 항목 | 내용 |
|------|------|
| 학습 스크립트 | `drive_train.py` |
| 모델 구조 | MobileNetV3-Small (ImageNet pretrained → fine-tune) |
| 입력 | 224×224 RGB 이미지 |
| 출력 | 키 조합 분류 (w / a / d / wa / wd 등) |
| 저장 모델 | `drive_best.pth` |

**핵심 설계 결정**
- **수평 flip 금지**: 좌/우 라벨이 반전되어 학습이 망가짐
- **밝기·대비 augmentation만 적용**: 조명 변화에 강건하게
- **클래스 가중치 자동 계산**: 직진(w) 데이터가 가장 많아 불균형이 심함 → 자동 보정

---

### 3단계 - 차선 세그멘테이션 학습

| 항목 | 내용 |
|------|------|
| 학습 스크립트 | `lane_train.py` |
| 모델 구조 | YOLOv8n-seg |
| 데이터 | `lane_dataset/` (LabelMe JSON으로 직접 라벨링) |
| 클래스 | inline(흰색 점선 0), outline(노란 실선 1), crosswalk(횡단보도 2) |
| 학습 결과 | `runs/lane_seg_v3/weights/best.pt` |
| 성능 | Mask mAP50 = 0.932 (v1 기준) |

**LabelMe → YOLO 자동 변환 파이프라인**
```
LabelMe JSON (폴리곤) ─► convert_json_to_yolo() ─► YOLO seg txt
                  + 자동으로 train/valid 8:2 분리
```

### 통합 테스트 v1 (모방학습 + 차선 안전장치)

- 실행 파일: `test_auto_drive(lane).py`

**차선 안전장치 동작 원리**

```
[별도 스레드]
  1) 매 프레임 흰색 점선(inline) 마스크 추출
  2) 화면 하단 절반에서 점선 x좌표 평균 계산
  3) 화면 중앙과의 오차(error) 계산
  4) |error| > 130px 이고 3프레임 연속이면 → 보정 명령 오버라이드

[메인 루프]
  모방학습 명령을 차선 보정 명령으로 덮어씀
```

> 별도 스레드 사용 이유: 차선 감지(GPU 추론)가 느려서 메인 루프를 막으면 안 됨

## 구현 2 : 횡단보도 인식 + 일시 정지

- 실행 파일: `test_auto_drive(lane)2.py`

차선 학습 데이터에 **crosswalk 클래스를 추가**해 재학습 → v3 모델 생성

**동작**
1. 차선 감지 스레드에서 crosswalk(class 2) 동시 감지
2. confidence > 0.45 이면 **3초 정지**
3. 정지 후 **30초 쿨다운** → 같은 횡단보도에서 중복 정지 방지

## 구현 3 : 장애물 회피 주행

- 실행 파일: `test_auto_drive(lane)3.py` (최종 버전)

### 장애물 감지 모델 학습

| 항목 | 내용 |
|------|------|
| 학습 스크립트 | `object_train.py` |
| 모델 구조 | YOLOv8n Detection |
| 원본 데이터 | `object_dataset/20260406_181559/` (LabelMe rectangle JSON) |
| 변환 데이터 | `object_train_dataset/` (자동 생성, train 80% / valid 20%) |
| 학습 결과 | `runs/object_det_v1/weights/best.pt` |

### 회피 기동 5단계

장애물이 감지되면 학습된 모델이 아닌 **직접 시간 기반 제어**로 회피합니다.

```
장애물 왼쪽 감지 → 오른쪽('d')으로 회피
장애물 오른쪽 감지 → 왼쪽('a')으로 회피

[1] 즉시 회피 조향  1.3초
[2] 직진           1.0초  ← 장애물 측면을 완전히 통과
[3] 복귀 조향      1.3~1.8초
[4] 직진           0.8초  ← 차선 안으로 복귀
[5] 쿨다운         4.0초  ← 재감지 방지
```

오감지 방지: 동일 방향으로 **N프레임 연속** 감지되어야 트리거
노이즈 제거: bbox 면적 < 2000px² 인 먼 물체는 무시

### 카메라 기반 회피의 한계

| 문제 | 이유 |
|------|------|
| 가까워질 때까지 감지가 어려움 | 카메라는 bbox가 커져야 신뢰도 ↑ |
| 회피 타이밍이 늦을 수 있음 | bbox가 충분히 커질 때 이미 가까운 상태 |
| **해결 방향** | **초음파 센서 추가** → 거리 기반 조기 감지 가능 |

- 차선 안전장치 상태 및 오차값

## 주요 트러블슈팅

| 문제 | 원인 | 해결 |
|------|------|------|
| W+A 동시 입력 안 됨 | cv2.waitKey()는 단일 키만 감지 | 키별 만료시각 독립 추적 |
| A/D 방향 반대 | 모터 배선 방향 | A↔D 모터값 스왑 |
| stop 데이터 과다 | 매 프레임 저장 시 90%가 stop | stop 프레임 저장 제외 |
| 차선 감지가 메인 루프 느리게 함 | YOLO 추론 시간 | 별도 스레드에서 처리 |
| 장애물 오감지 빈번 | 단일 프레임 노이즈 | N프레임 연속 감지 필터링 |
| 장애물 회피 후 차선 이탈 | 단순 조향만으로는 부족 | 5단계 기동 시퀀스 도입 |
| 먼 물체도 감지 | bbox 크기 필터 없음 | bbox 면적 < 2000px² 무시 |
| OMP 중복 오류 | Windows + conda 환경 충돌 | KMP_DUPLICATE_LIB_OK=TRUE |
